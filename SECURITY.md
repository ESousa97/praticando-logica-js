| Versão | Suportada          |
| ------ | ------------------ |
| 2.1.x  | :white_check_mark: |

## Reportando uma Vulnerabilidade

A segurança do Praticando Lógica JS é levada a sério. Se você descobrir uma vulnerabilidade de segurança, por favor siga as instruções abaixo.

### Como Reportar

**NÃO** abra uma issue pública para vulnerabilidades de segurança.

Em vez disso, envie um e-mail para: **enoquesousa97@outlook.com**

Inclua as seguintes informações:

1. **Descrição**: Uma descrição clara da vulnerabilidade
2. **Passos para reproduzir**: Passos detalhados para reproduzir o problema
3. **Impacto**: Qual o impacto potencial da vulnerabilidade
4. **Versão afetada**: Qual versão do projeto está afetada
5. **Possível solução**: Se você tiver uma sugestão de correção

### O que Esperar

- **Confirmação**: Responderemos em até 48 horas confirmando o recebimento
- **Avaliação**: Avaliaremos a vulnerabilidade em até 7 dias
- **Correção**: Trabalharemos em uma correção o mais rápido possível
- **Divulgação**: Coordenaremos a divulgação pública após a correção

### Práticas de Segurança do Projeto

Este projeto implementa as seguintes práticas de segurança:

- **Dependabot**: Atualizações automáticas de dependências vulneráveis
- **CI/CD**: Verificações automatizadas em cada PR
- **Linting**: ESLint com regras de segurança
- **Revisão de código**: Todas as mudanças são revisadas antes do merge

### Escopo

Esta política se aplica ao código-fonte do repositório principal. Vulnerabilidades em dependências devem ser reportadas aos respectivos mantenedores.

## Boas Práticas para Contribuidores

- Nunca commite secrets, tokens ou credenciais
- Mantenha dependências atualizadas
- Siga as práticas de código seguro
- Revise mudanças de segurança cuidadosamente

## Agradecimentos

Agradecemos a todos que reportam vulnerabilidades de forma responsável. Sua contribuição ajuda a manter o projeto seguro para todos.
